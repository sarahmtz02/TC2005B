Cuestionario


1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es la conexión entre dispositivos (computadoras, celulares, smartwatches, etc.), mientras que la Web es la colección de sitios y páginas dentro del mismo internet. En otras palabras, utilizamos el internet (que conecta los dispositivos) para acceder a todas las páginas que hay dentro de la Web.


2. ¿Cuáles son las partes de un URL?

- Protocolo: es el formato en el que se podrá intercambiar información entre dispositivos. Hay múltiples tipos, por ejemplo, http (HyperText Transfer Protocol) que funciona para la visualización de páginas web.

- Servidor: es el nombre de la página a la que se accede y toma en cuenta el nombre del servidor y el dominio. El dominio puede tener terminaciones técnicas como .net (network), .gob (gobierno), .com (commercial) o incluso tener terminaciones relacionadas con la región geográfica como .mx (México), .uk (United Kingdom), entre otras.

- Puerto: es un número que indica la entrada a la máquina de la que vamos a obtener la información. Suele ir de mano en mano con el protocolo. La URL no funciona si falta alguno de los 2. Sin embargo, con algunas páginas no es necesario ponerlo ya que, si tiene protocolo http, suele ir conectado al puerto 80 y se conecta automáticamente.

- Directorio: Al igual que un directorio normal, permite a la máquina saber dónde se encuentra la página que estamos buscando. Sirve para organizar las páginas y poder ubicar las que se necesitan con facilidad. ¡Es como un índice!

- Página: Contiene el contenido (valga la redundancia) de la página web. Expone concretamente lo que se encontrará en esa página en específico.


3.¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

- GET: se ocupa para recuperar un dato en específico.

- HEAD: muy similar al método GET, pero ignora el cuerpo de la respuesta.

- POST: realiza cambios de estado dentro del servidor. Se utiliza para colocar una entidad en un recurso concreto.

- PUT: reemplaza el contenido de un recurso con los datos que vienen con el comando.

- PATCH: modifica parcialmente un recurso para reemplazar con otros datos.

- DELETE: se utiliza para borrar un recurso en específico.


4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Para esta situación, se podría utilizar el método GET. El método GET, como se mencionó arriba, se usa para solicitar al servidor que envíe un recurso de regreso, como una respuesta, pero esta respuesta es un cuerpo vacío, lo que envía los datos a la URL y así se pueden pasar estos datos al servidor.


5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se podría usar el método PUT, ya que buscamos reemplazar la “página de inicio” (por ejemplo, si usamos Google Chrome, sería reemplazar la página inicial del buscador Google con la nueva URL) con la página a la que deseamos acceder.


6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Si te encuentras como respuesta el código 200, ¡no te preocupes! No ha habido ningún error. Es sólo el código 200 OK. Quiere decir que todo se ha desarrollado correctamente y con éxito.


7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

¡Ha ocurrido un error! Parece que el recurso que buscabas no se encuentra en el servidor. El código 404 suele asociarse con vínculos rotos o vínculos muertos, lo cual quiere decir que el recurso se ausenta de manera temporal o permanente del servidor y que el desarrollador debe echarle un vistazo a la situación.


8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

¡Oops! Parece que el servidor ha tenido problemas y no se puede completar tu solicitud. Usualmente ocurre cuando no se puede encontrar una mejor respuesta que arrojar, pero no ataques a tu desarrollador porque creo que no es su culpa (bueno, depende).

10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Responderé esta pregunta con un pequeño resumen de una parte del blog de Brenda Barrón (2021).

El primer detalle que ella vio fue la compatibilidad de los sitios con las diferentes etiquetas de HTML4 y HTML5. Observó que este fue un gran problema al principio, ya que los sitios con HTML4 se podían visualizar en el navegador Internet Explorer, pero fallaban cuando se abrían en navegadores más modernos, mientras que las funciones con HTML5 funcionaban y eran compatibles con muchos navegadores que se usan más comúnmente ahora.

Comenta también del avance con HTML5 con los casos de soporte multimedia, ya que es muy sencillo implementar algún archivo de sonido, imagen o video, mientras que con HTML4 se requería el uso de JavaScript.


11. ¿Qué componentes de estructura y estilo tiene una tabla?

Bueno, dirigiéndonos a los básicos, una tabla es una herramienta visual que nos ayuda a exponer información en filas y columnas dependiendo de la relación que tienen entre sí. Permite organizar datos (texto, imágenes, vínculos, entre otros). Cada tabla puede tener una pequeña descripción como encabezado explicando la información que se encuentra en ella. Cada celda es un espacio para almacenar esta información y se pueden ver como “coordenadas” para ver con qué fila y columna se relacionan.

Preguntas Laboratorio 3


1. Como ingeniero de software, ¿cuál es tu recomendación de !important en un CSS?

Cuando usamos la declaración !important en nuestro archivo de estilo CSS, indicamos que ciertas propiedades lleven preferencia ante otras declaradas dentro del mismo. Sin embargo, siendo una declaración nueva, varios navegadores antiguos pueden tener complicaciones cuando lo manejan, por lo que hay que estar trabajando con un navegador compatible con la función. En mi opinión, me parece una declaración de apoyo en un archivo CSS. Puede evitar que se mezclen las cosas, pero no es para ponerse en cada línea de código ni en cada detalle del archivo.


2. Si se pone una imagen de fondo en un HTML, ¿por qué debe elegirse con cuidado?

Porque, al igual que con esta página, si la imagen no tiene el tamaño para cubrir la página, se multiplica para cubrir todo el fondo. Entonces debemos ser cuidadosos para que no se vea demasiado evidente.


3. Como ingeniero de software, ¿cuál es tu mejor recomendación al elegir las unidades de una propiedad de estilo entre %, px y pt?

Cuando se trata de imágenes, definitivamente prefiero el control de px (píxeles). La pantalla de una computadora tiene un cierto número de píxeles para cubrir, y si aseguramos que la imagen tenga la misma cantidad de píxeles, entonces aseguramos de que se pueda cubrir el espacio deseado con la imágen.


4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento de tu sitio?

Cuando estamos trabajando con un código más extenso, el CSS crece junto con él. Usualmente, los documentos de CSS suelen ser enormes, así que pueden retrasar el desempeño de la página Web. Con la versión minimizada, podemos hacer que el CSS pese menos y sea más fácil de leer.
